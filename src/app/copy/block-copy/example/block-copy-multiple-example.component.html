Multiple same component example

You can include as many block copy components you want on a single page. The nature of this example affords an opportunity to point out a potentional pitfall - not properly constructing accessible names. With this component, you have full control over the text displayed in the UI vs what's announced by your supported screen reader.

How this component handles accessibility

This example is exposed in the VoiceOver rotor menu as a Form Control in the format of "Copy {{'{'}}accessibleName{{'}'}} button". This ensures that context for the copy button is provided across multiple navigation formats.
